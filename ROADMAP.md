# 🗺 LoadLink — Development Roadmap

## Overview

LoadLink is being built in 4 phases, designed to get to market fast and iterate based on real user feedback.

---

## ✅ Phase 1 — MVP (Complete)

**Timeline:** 4-8 weeks | **Status:** Done

The core marketplace — prove the concept works.

- [x] Landing page with role selection (Business / Driver)
- [x] Business booking flow (4-step wizard)
  - [x] Job details form (what, where, when)
  - [x] Vehicle selection with recommendations
  - [x] Service tier pricing (Economy / Standard / Express)
  - [x] Booking confirmation with driver assignment
- [x] Driver dashboard
  - [x] Load board with filtering (All / Fixed Price / Quotes)
  - [x] Fixed price instant accept (Uber-style)
  - [x] Quote submission with rate guidance
  - [x] Active jobs with progress tracking
  - [x] Driver availability posting
  - [x] Available drivers browse view
- [x] 3 booking modes working
  - [x] Mode 1: Quote Request
  - [x] Mode 2: Fixed Price / Instant Accept
  - [x] Mode 3: Driver Availability Post
- [x] Special requirements (tail lift, fragile, 2-man, temp controlled)
- [x] 6 vehicle categories with payload info
- [x] Mobile-responsive design

## 📊 Priority Framework

We prioritise features based on:

1. **Revenue impact** — Does it help us make money or save users money?
2. **User demand** — Are people asking for it?
3. **Competitive advantage** — Does it differentiate us from CX?
4. **Build effort** — How long does it take vs the impact?
